# Micro Rover Project

## Overview
The Micro Rover project is a modular and extensible software framework designed for controlling and simulating a micro rover. It includes various components for differential drive control, keyboard control, IMU-based control, and web-based control. The rover is a 2WD system built on the ESP8266 microcontroller and is controlled through a webpage interface. The project is structured to support development, testing, and deployment of rover functionalities.

## Project Structure

### Root Directory
- **index.html**: Controller Web Page

### `build/`
Contains build-related files and directories generated by the `colcon` build system.

- **COLCON_IGNORE**: Marker file to ignore certain directories during the build process.
- **rover/**: Contains build artifacts for the `rover` package.

### `imu_control/`
- **main.py**: Script for controlling the rover using IMU (Inertial Measurement Unit) data.

### `install/`
Contains installation scripts and setup files for the project.

- **setup.* files**: Scripts for setting up the environment for the rover package.
- **rover/**: Installed files for the `rover` package, including Python modules and metadata.

### `keyboard_control/`
- **main.py**: Script for controlling the rover using keyboard inputs.

### `log/`
Contains logs generated during the build and runtime.

- **latest**: Symlink to the latest log directory.
- **build_*/**: Timestamped directories containing build logs.

### `rover_differential_drive/`
- **rover_differential_drive.ino**: Arduino sketch for differential drive control.

### `rover_drive.ino/`
- **rover_drive.ino.ino**: Arduino sketch for basic rover drive functionality.

### `rover_path_tracer/`
- **rover_path_tracer.ino**: Arduino sketch for path tracing functionality. This code is intended to be uploaded to the ESP8266 microcontroller.

### `rover_web_control/`
- **rover_web_control.ino**: Arduino sketch for web-based rover control.

### `src/`
Contains the source code for the `rover` package.

- **rover/**: Python package for the rover, including:
  - **`__init__.py`**: Package initialization.
  - **`drive.py`**: Core functionality for driving the rover.
- **test/**: Unit tests for the `rover` package.
  - **test_copyright.py**: Copyright compliance tests.
  - **test_flake8.py**: Code style tests using Flake8.
  - **test_pep257.py**: Docstring style tests using PEP 257.
- **setup.py**: Setup script for the `rover` package.
- **setup.cfg**: Configuration file for the `rover` package.
- **package.xml**: ROS package metadata.

## Getting Started

### Prerequisites
- Python 3.10 or higher
- Arduino IDE (for `.ino` files)

### Installation
1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd micro-rover
   ```

### Uploading Code to ESP8266
1. Open the Arduino IDE.
2. Load the `rover_path_tracer/rover_path_tracer.ino` file.
3. Select the appropriate board (ESP8266) and port from the Tools menu.
4. Upload the code to the ESP8266.

### Running the Rover
- **Keyboard Control**:
  ```bash
  python keyboard_control/main.py
  ```
- **IMU Control**:
  ```bash
  python imu_control/main.py
  ```

### Testing
Run the unit tests:
```bash
python -m unittest discover
```
